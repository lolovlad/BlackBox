# BlackBox на устройстве (Linux): пошаговая инструкция

Сделайте шаги **по порядку**. Если что-то не получилось — остановитесь на этом шаге и исправьте ошибку (текст в терминале обычно говорит, чего не хватает).

---

## Что должно быть заранее

- Устройство с **Linux** (например Raspberry Pi или промышленный контроллер).
- Вы умеете зайти на него по **SSH** или открыть на нём терминал.
- Проект BlackBox уже **скопирован** на устройство в какую-нибудь папку (например `/home/pi/BlackBox` или `/opt/blackbox`).

---

## Шаг 1. Установить `uv` (менеджер Python и зависимостей)

В терминале на устройстве выполните **одну** команду с официальной инструкции:

- Сайт: [https://docs.astral.sh/uv/getting-started/installation/](https://docs.astral.sh/uv/getting-started/installation/)

Проверка:

```bash
uv --version
```

Если версия показалась — шаг выполнен.

---

## Шаг 2. Перейти в папку проекта

Подставьте **свой** путь к проекту:

```bash
cd /путь/к/BlackBox
```

Проверка: в этой папке должны быть файлы `pyproject.toml` и папка `src`.

---

## Шаг 3. Создать файл `.env` (настройки приложения)

**Вариант А — мастер в терминале** (удобно, если вы подключены к устройству с клавиатурой или по SSH с «живым» терминалом):

```bash
chmod +x scripts/linux/create_env.sh
sh scripts/linux/create_env.sh
```

Отвечайте на вопросы или нажимайте Enter, чтобы оставить значение по умолчанию.

**Вариант Б — без вопросов** (если терминала «настоящего» нет, например скрипт идёт без TTY): тот же скрипт сам запишет **разумные значения по умолчанию**:

```bash
sh scripts/linux/create_env.sh
```

**Вариант В — вручную:** скопируйте пример из корневого `README.md` в файл `.env` в **корне проекта** и отредактируйте пути и пароли.

Проверка:

```bash
test -f .env && echo OK
```

Должно напечатать `OK`.

---

## Шаг 4. Установить зависимости проекта

```bash
uv sync --frozen --no-dev
```

Дождитесь окончания без ошибок.

---

## Шаг 5. Указать Flask, где приложение (для миграций)

```bash
export FLASK_APP=src.web_app:app
```

(Можно добавить эту строку в `~/.profile`, если не хотите вводить каждый раз.)

---

## Шаг 6. Применить миграции базы данных

```bash
uv run flask db upgrade
```

---

## Шаг 7. Создать первых пользователей веб-интерфейса

```bash
DISABLE_MODBUS_COLLECTOR=1 uv run python -m src.seed
```

По умолчанию часто будут логины `admin` / `admin` и `user` / `user` (если вы не меняли их в `.env`).

---

## Шаг 8. Запустить веб-приложение **вручную** (в этом окне терминала)

Это **обычный запуск на переднем плане**: пока команда работает, сайт доступен; остановка — **Ctrl+C**.

```bash
chmod +x scripts/linux/run_blackbox.sh
./scripts/linux/run_blackbox.sh
```

Либо те же действия одной цепочкой без скрипта (если хотите сами):

```bash
set -a && . ./.env && set +a
uv sync --frozen --no-dev
uv run flask db upgrade
uv run uvicorn src.web_app:app --host "${HOST:-0.0.0.0}" --port "${PORT:-5000}" --interface wsgi --log-level info --access-log
```

Откройте в браузере (с другого компьютера в сети подставьте **IP устройства**):

```text
http://IP-УСТРОЙСТВА:5000/
```

Порт `5000` — если в `.env` не задали другой `PORT`.

---

## Важно: два разных смысла «фона»

1. **Ручной запуск** (`run_blackbox.sh` или `uvicorn` в терминале) — процесс **привязан к этому окну**. Закрыли SSH или нажали Ctrl+C — приложение остановилось. Это нормально для проверки и отладки.

2. **Автозапуск при включении устройства** — это **отдельное** действие: его делает **systemd** (служба ОС), а не «запуск в фоне» из того же скрипта. Подробно, без смешивания с ручным запуском: файл **`LINUX_AUTOSTART.md`**.

Кратко для автозапуска (нужны права root):

```bash
chmod +x scripts/linux/install_systemd_service.sh
sudo sh scripts/linux/install_systemd_service.sh
```

Скрипт копирует проект в `/opt/blackbox`, создаёт пользователя `blackbox`, при отсутствии `.env` создаёт его дефолтами, ставит и запускает службу `blackbox.service`.

## Частые проблемы

| Симптом | Что сделать |
|--------|-------------|
| «`.env` не найден» | Шаг 3 — файл должен лежать в **корне проекта**, имя ровно `.env`. |
| «`uv` not found» | Шаг 1. |
| Не открывается сайт с другого ПК | Проверьте `HOST=0.0.0.0` в `.env`, файрвол, что порт тот же, что в `PORT`. |
| Ошибки миграций | Шаг 5 (`FLASK_APP`), затем снова шаг 6. |
