Имена переменных должны совпадать с полями name в settings.json (включая имена с точками, например Gov.Reg.Value). Для дискретов и аналогов — те же имена; для ошибок в in — строки из секций bits у bitfield. Допустимы функции: abs, min, max, round, len.

Как писать правила аварий

Правило должно возвращать True или False.

  • Проверка активной аварии по тексту: '1670 EmergStop DI24' in active_alarms
  • Комбинация с аналогом: ('Start Fail' in active_alarms) and (RPM < 300)
  • Комбинация с дискретом: ('1770 Low Oil Press. DI92' in active_alarms) and Engine_running
  • Логические операторы: and, or, not
  • Разрешенные функции: abs, min, max, round, len

Имена переменных берите из settings.json (поле name). Для in active_alarms используйте точный текст аварии из секций bits.
